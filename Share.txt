[이 파일의 용도]
윈도우 노트북과 맥북, 두 환경에서 이 과제를 이어서 작업하기 위한 진행상황 공유 파일.
Claude와의 대화 세션은 두 노트북 간에 공유되지 않으므로, 각 단계가 "확정/완료"될 때마다
이 파일을 갱신한다. 다른 노트북에서 작업을 시작할 때는 먼저 이 파일을 읽고 이어서 진행한다.
(git add / commit / push, 다른 쪽에서는 git pull 로 동기화)

===================================================
진행 방식 (합의됨)
===================================================
Goal.txt의 미션 순서대로 "단계별"로 진행. 각 단계가 끝나면 사용자 확인 후 다음 단계로 이동.
단계 순서:
  1. 7개 HTML 파일 기본 구조 (필수 시맨틱 태그)
  2. 미션1~2: CSS 분리, 폰트/색상, container, 카드/테이블 스타일
  3. 미션3~4: Flexbox/Grid 반응형 레이아웃
  4. 미션5: hover/transition/페이드인 애니메이션
  5. 미션6~8: JS (upDown, grade, bag)
  6. 미션9: 날씨 API (weatherAPI.js, realtimeInfo.js)

===================================================
현재 상태
===================================================
[완료] 1단계 - 7개 HTML 파일 기본 구조
[대기] 2단계 - 미션1~2 (CSS)
[대기] 3단계 - 미션3~4 (반응형 레이아웃)
[대기] 4단계 - 미션5 (애니메이션)
[대기] 5단계 - 미션6~8 (JS 게임/성적/가방)
[대기] 6단계 - 미션9 (날씨 API)

===================================================
지금까지 내린 주요 결정 (다음 단계에서 지켜야 할 컨벤션)
===================================================
- 사이트 제목/컨셉: "나의 포털" (개인 이름 대신 임시로 이렇게 정함. 원하면 나중에 교체)
- 폴더 구조는 Teacher_Front와 동일하게 루트 기준 형제 폴더로 구성:
    /html  - 7개 html 파일
    /css   - style.css (아직 미생성, 2단계에서 생성 예정)
    /script - upDown.js, grade.js, bag.js, weatherAPI.js, realtimeInfo.js (아직 미생성)
    /media - myTrip.html용 이미지/오디오/비디오
  html 파일에서는 ../css/, ../script/, ../media/ 형태의 상대경로로 참조.
- nav 메뉴 순서: myClass, myHoliday, myProfile, myTrip, signUp (Goal.txt 명세 순서를 따름).
  index.html에는 5개 전부, 각 페이지에는 자기 자신을 뺀 나머지 + index(Home) 링크.
- myClass.html은 명세상 "CSS 사용 금지" 조건이 있어서 1단계에서는 <link rel="stylesheet">를
  넣지 않음. 미션2에서 "table, th, td를 꾸며 깔끔한 테이블 생성"을 요구하므로, 그때 가서
  CSS를 연결하기로 함 (교사 의도상 처음엔 순수 HTML 표, 이후 스타일링 단계에서 CSS 적용).
- myClass.html 시간표는 임의로 구성한 부트캠프 스타일 시간표 (09:00~18:00, 월~금).
  rowspan으로 2시간 수업(월요일 09-11시 HTML기초, 매일 14-16시 실습/프로젝트) 표현,
  colspan으로 점심시간(12-13시) 표현.
- myProfile.html 내용(좋아하는 음식/올해 할 일/나를 설명하는 단어)은 임의로 자연스럽게 작성.
- myTrip.html 여행지는 임시로 "제주도, 부산, 강릉" 3곳 사용 (나중에 사용자가 실제 여행지로 교체 가능).
  → 3곳으로 만든 이유: 3단계(미션3)에서 Grid 3열 배치를 적용할 예정이라 개수를 맞춰둠.
- media 폴더의 이미지/오디오/비디오는 ffmpeg로 만든 "진짜 placeholder"
  (단색 배경 이미지 3장, 짧은 톤 오디오, 짧은 컬러 비디오). 실제 사진/영상으로 교체 예정.
  파일명: media/place1.jpg, place2.jpg, place3.jpg, trip-audio.mp3, trip-video.mp4
- signUp.html: action="signUpResult.html" method="get" (명세 그대로).
- Teacher_Front 폴더는 구조 참고용으로만 보고, 디자인/문구를 그대로 베끼지 않기로 함
  (Goal.txt 참고 파일 안내에 따른 원칙).

===================================================
다음에 할 일 (다음 세션이 여기서부터 이어감)
===================================================
- 2단계 시작: /css/style.css 생성
  - 구글 폰트 적용 (아직 폰트 미선정 - 다음 세션에서 정하거나 사용자에게 확인)
  - body 전체 줄간격/색/배경색
  - h1, h2 꾸미기 (색, 크기, padding, border)
  - 링크 스타일 (color, text-decoration)
  - 모든 html에 <link rel="stylesheet" href="../css/style.css"> 연결 확인
  - body 아래 <div class="container"> 추가하고 가운데 정렬
  - myTrip.html의 <p class="trip-card"> 카드 스타일링
  - myClass.html도 이 단계에서 <link> 추가하고 table/th/td 스타일링

===================================================
업데이트 규칙
===================================================
각 미션 단계가 사용자 확인을 받고 "확정"되면, 이 파일의 "현재 상태" 표시를 갱신하고
"주요 결정" 섹션에 새로 정해진 컨벤션을 추가한 뒤, git commit/push 할 것.
새 노트북에서 시작할 때는 이 파일을 먼저 읽고 "다음에 할 일"부터 이어서 진행.
